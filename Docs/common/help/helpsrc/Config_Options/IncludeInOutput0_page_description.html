HELP - IncludeInOutput
HelpConfig OptionsIncludeInOutput
Description  
Specify what is included in the output.
CREATE_DOC_FOR_FILESIf set to YES DoxyS will create Documentation for files. Documentation for files features included files and included by diagrams, if INCLUDE_GRAPH and INCLUDED_BY_GRAPH are set to YES.
Default value:
YES
CREATE_SOURCECODE_FOR_FILESIf the CREATE_SOURCECODE_FOR_FILES tag is set to YES and CREATE_DOC_FOR_FILES is set to YES DoxyS will create the tab 'Source' that holds the sourcecode for the hole file.
Default value:
NO
CREATE_SOURCECODE_FOR_MEMBERSIf the CREATE_SOURCECODE_FOR_MEMBERS tag is set to YES DoxyS will create the tab 'Source' that holds the sourcecode for the member.
Default value:
YES
EXTRACT_PRIVATEIf set to YES all private members of a class will be included in the documentation.
Default value:
YES
EXTRACT_PROTECTEDIf the EXTRACT_PROTECTED tag is set to YES all protected members of a class will be included in the documentation.
Default value:
YES
EXTRACT_STATICIf the EXTRACT_STATIC tag is set to YES all static members of a file will be included in the documentation.
Default value:
YES
EXTRACT_LOCAL_CLASSESIf the EXTRACT_LOCAL_CLASSES tag is set to YES classes (and structs) defined locally in source files will be included in the documentation. If set to NO only classes defined in header files are included.
Default value:
YES
HIDE_UNDOC_NAMESPACESIf the HIDE_UNDOC_NAMESPACES tag is set to YES, DoxyS will hide all undocumented namespaces.
Default value:
NO
HIDE_UNDOC_CLASSESIf the HIDE_UNDOC_CLASSES tag is set to YES, DoxyS will hide all undocumented classes that are normally visible in the class hierarchy. If set to NO (the default) these classes will be included in the various overviews.
Default value:
NO
HIDE_UNDOC_STRUCTSIf the HIDE_UNDOC_STRUCTS tag is set to YES, DoxyS will hide all undocumented structs that are normally visible in the hierarchy. If set to NO (the default) these structs will be included in the various overviews.
Default value:
NO
HIDE_UNDOC_UNIONSIf the HIDE_UNDOC_UNIONS tag is set to YES, DoxyS will hide all undocumented unions that are normally visible in the hierarchy. If set to NO (the default) these unions will be included in the various overviews.
Default value:
NO
HIDE_UNDOC_FUNCTIONSIf set to YES, DoxyS will hide all undocumented functions inside classes, files or namespaces.
Default value:
NO
HIDE_UNDOC_VARIABLESIf set to YES, DoxyS will hide all undocumented variables inside classes, files or namespaces.
Default value:
NO
HIDE_UNDOC_ENUMSIf set to YES, DoxyS will hide all undocumented enumerations inside classes, files or namespaces.
Default value:
NO
HIDE_UNDOC_ENUM_VALUESIf set to YES, DoxyS will hide all undocumented enum valuesinside classes, files or namespaces.
Default value:
NO
HIDE_UNDOC_TYPEDEFSIf set to YES, DoxyS will hide all undocumented typedefs inside classes, files or namespaces.
Default value:
NO
HIDE_UNDOC_MACROSIf set to YES, DoxyS will hide all undocumented defines.
Default value:
NO
HIDE_UNDOC_DEFINESIf set to YES, DoxyS will hide all undocumented defines.
Default value:
YES
GENERATE_TODOLISTThe GENERATE_TODOLIST tag can be used to enable (YES) or disable (NO) the todo list. This list is created by putting odo commands in the documentation.
Default value:
YES
GENERATE_BUGLISTThe GENERATE_BUGLIST tag can be used to enable (YES) or disable (NO) the bug list. This list is created by putting bug commands in the documentation.
Default value:
YES
GENERATE_DEPRECATEDLISTThe GENERATE_DEPRECATEDLIST tag can be used to enable (YES) or disable (NO) the deprecated list. This list is created by putting deprecated commands in the documentation.
Default value:
YES
